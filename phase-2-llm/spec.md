# Phase 2: LLM Provider - Спецификация

## Цель фазы

Реализовать поддержку разных LLM провайдеров (Ollama, OpenAI).

## Требования

### LLM-001: Абстрактный провайдер
Система ДОЛЖНА иметь абстрактный класс LLMProvider.

### LLM-002: Ollama провайдер
Система ДОЛЖНА поддерживать Ollama с локальными моделями.

### LLM-003: OpenAI провайдер
Система ДОЛЖНА поддерживать OpenAI API.

### LLM-004: Chat интерфейс
Система ДОЛЖНА иметь единый интерфейс для чата.

### LLM-005: Tools support
Система ДОЛЖНА поддерживать вызов инструментов через LLM.

## Выходные артефакты

- src/llm/base.py — абстрактный класс
- src/llm/ollama.py — Ollama провайдер
- src/llm/openai.py — OpenAI провайдер
- Тесты
